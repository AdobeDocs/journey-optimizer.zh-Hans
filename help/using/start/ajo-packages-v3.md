# Adobe Journey Optimizer包和功能 {#ajo-packages}

[!DNL Adobe Journey Optimizer]使用模块化打包模型。 从与您的主要编排用例匹配的基本选件开始，然后添加您需要的渠道和高级功能。

>[!NOTE]
>
>功能可用性可能因您的许可协议、选定的插件、区域可用性、渠道配置和实施先决条件而异。 请联系您的Adobe代表，以了解特定于贵组织的详细信息。

## 第1步 — 选择基本选件 {#base-offers}

提供了三个基本选件。 选择最符合您与客户主要互动方式的。

| 基本选件 | 最适合 | 其中包含的内容 |
|-----------|---------|-----------------|
| **Journey Optimizer — 营销活动** | 批次、营销人员计划外联 | 基于受众、计划的活动编排，用于批量参与和营销人员计划的外联。 |
| **Journey Optimizer -历程** | 实时客户参与 | 事件驱动的1:1历程编排，用于实时客户参与。 |
| **Journey Optimizer — 营销活动和历程** | 同时需要两者的客户 | 在一个选件中包含基于受众的活动编排和实时历程编排。 |

### 促销活动与历程 — 有什么区别？ {#campaigns-vs-journeys}

| 功能 | Journey Optimizer — 营销活动 | Journey Optimizer -历程 | Journey Optimizer — 促销活动和历程 |
|--|:-----------------------------:|:----------------------------:|:----------------------------------------:|
| 基于受众的批量编排 | ✓ | — | ✓ |
| 实时事件驱动编排 | — | ✓ | ✓ |
| 可用的渠道加载项 | ✓ | ✓ | ✓ |
| Decisioning加载项可用 | ✓ | ✓ | ✓ |

### 何时应选择营销活动？ {#when-to-choose-campaigns}

如果您的主要用例为：****

- 计划的活动
- 基于受众的外联
- 批量营销工作流
- 营销人员计划的参与计划

### 何时应选择历程？ {#when-to-choose-journeys}

如果您的主要用例为：，请选择&#x200B;**Journey Optimizer -历程**

- 事件触发式通信
- 事件后跟进流程
- 入门和生命周期历程
- 实时1:1参与

### 何时应选择“促销活动和历程”？ {#when-to-choose-both}

选择&#x200B;**Journey Optimizer — 促销活动和历程**（如果您同时需要）：

- 基于受众的批量编排，以及
- 实时事件驱动编排

## 常见问题 {#faq}

+++**每个基本选件是否包含每个渠道？**

没有。 基本选件定义了您的编排功能。 渠道访问取决于贵组织的插件和许可配置。

+++

+++**促销活动与历程之间有何区别？**

**营销活动**&#x200B;基于受众，是营销人员计划的。 它们最适合计划外联、促销项目和批量活动工作流。

**历程**&#x200B;是实时事件驱动的。 它们最适合1:1参与对客户行为产生反应。

**营销活动和历程**&#x200B;同时包含两个编排模型。

+++

+++**编排的营销活动支持哪些渠道？**

编排的活动支持用于活动编排工作流的渠道子集。 渠道可用性取决于您的许可配置。 [了解关于编排的营销活动的更多信息](../orchestrated/gs-orchestrated-campaigns.md)

+++

+++**Decisioning是否包含在每个配置中？**

没有。 Decisioning是一个单独的高级功能加载项。 [了解如何使用决策](../experience-decisioning/gs-experience-decisioning.md)

+++

+++**我听说过Select、Prime或Ultimate。 这些是否仍是当前的打包模型？**

[!DNL Adobe Journey Optimizer]现在通过基于基本选件和插件构建的模块化打包模型提供。 如果您是使用旧版打包术语的现有客户，请联系您的Adobe代表以确认您当前的授权。

+++
