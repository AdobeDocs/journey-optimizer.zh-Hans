# 角色和责任

>[!BEGINSHADEBOX]

**在此页面上：**&#x200B;了解Adobe Journey Optimizer实施中的关键角色及其职责，以便您能够为团队找到正确的起点和快速入门任务。

>[!ENDSHADEBOX]

Adobe Journey Optimizer 助力品牌在客户历程全程传递互联、情境化且个性化的体验。 Journey Optimizer 以端到端的方式聚焦于规模、速度与灵活性，将三大核心价值驱动因素融合于统一应用中：

* 由 Adobe 实时客户轮廓驱动的&#x200B;**实时客户洞察与参与度**
* **现代化全渠道编排**，通过统一画布实现实时历程与批量营销活动的管理，并搭配现代消息设计器
* **智能决策与个性化**，借助决策管理与 AI/ML 功能实现

Journey Optimizer 提供两种主要方法来触达和吸引客户：

* **历程** – 实时、一对一的编排，每位客户按自己的节奏推进，由行为或事件触发。 最适合用于新用户引导流程、购物车弃购以及生命周期内的互动。
* **营销活动** – 基于受众的消息传递，有三种投放模式，具体取决于您的用例：
   * **操作营销活动** – 按计划或周期向指定受众群发的消息。 最适合用于新闻稿、促销公告和产品发布。
   * **API 触发的营销活动** – 由外部系统通过 API 触发的按需消息。 最适合用于订单确认、发货提醒和帐户通知等事务性消息。
   * **编排的营销活动** – 复杂的批量工作流，支持多实体分段和基于画布的执行。 最适合季节性促销、多步骤批量项目和需要精确预发送量的营销活动。

这种统一的体验让您能够在同一平台上完成整个用例的实施，从定义受众、设计历程，到创建个性化内容及分析结果。 本文档介绍了高效利用 Journey Optimizer 时所涉及的关键角色及其对应职责，并提供了快速入门指南。

**重要说明：** Adobe Journey Optimizer 为不同角色设定了明确的职责。 根据组织架构的不同，可由单人兼任多个或全部角色。

## 开始之前 {#before-you-begin}

成功的实施始于充分的准备。 在配置 Journey Optimizer 之前，请确保您的团队就以下事项达成一致：

* **首先定义您的用例** – 确定您将要处理的客户场景并为它们设定优先级。 这指导着从[数据管理](../data/gs-data.md)到[渠道设置](../configuration/get-started-configuration.md)的每个配置决策。
* **让所有触及客户体验的团队都参与进来** – Journey Optimizer 的实施通常涵盖营销、IT、数据和运营职能领域。 团队之间尽早对齐共识，可以避免后续返工。
* **建立共享的客户标识符** – 商定所有数据源中都存在的通用标识符（如 CRM ID 或电子邮件地址）。 这是[统一客户轮廓](../audience/get-started-profiles.md)的基础。
* **验证数据隐私合规性** – 在执行摄取之前，确保您计划连接的所有数据源都符合适用的[隐私法规](../privacy/get-started-privacy.md)。
* **在上线之前制定测试计划** – 在开发或暂存沙盒中，验证[事件触发器、历程条件和渠道操作](../building-journeys/journey-gs.md)是否按预期运行。
* **准备您的品牌内容和资产库** – 确定您的团队将在历程和营销活动中使用的数字资产、模板和品牌准则。 在启动之前，将它们加载到 Journey Optimizer 的[内置资产库](../integrations/assets.md)中，这样可加快消息创建速度，从一开始就确保品牌一致性。

## 基于角色的快速入门指南

为简化实施流程，Adobe Journey Optimizer 根据专业领域将任务划分给特定角色。 每个角色专注于提供无缝客户体验所需的基本任务。

| 角色 | 主要职责 | 关键技能 | 典型任务 |
|-------------------|----------------------------------|--------------------------------|-----------------------------------------------|
| **管理员** | 环境设置和访问管理 | 系统配置、用户管理、安全性 | 设置沙盒，管理用户权限，配置渠道和消息预设 |
| **数据工程师** | 客户轮廓数据与数据源 | 数据建模、 XDM 架构、源连接器 | 将轮廓与业务数据建模为架构，配置源连接器，监测数据摄取 |
| **开发人员** | 技术实施和集成 | 移动端/网页 SDK、API、事件驱动型架构 | 集成 SDK、实施事件、构建自定义操作端点 |
| **营销人员** | 历程设计和个性化体验 | 历程编排、内容创建、受众目标选择 | 设计客户历程，创建并个性化消息，管理优惠和决策组件，定义受众群体 |

每个角色负责 Adobe Journey Optimizer 实施的特定阶段工作，并确保部署流程结构清晰、高效推进。

## 实现顺序和角色依赖关系

成功的 Journey Optimizer 实施通常遵循以下顺序，该顺序体现了角色之间的依赖关系：

1. **管理员**：配置环境\
   管理员通过配置沙盒环境、设置访问控制及准备渠道配置来奠定基础。 此项工作必须首先完成，以便其他团队能够开展工作。
   * 配置开发、暂存和生产沙盒
   * 设置角色、权限和对象级访问控制 (OLAC)
   * 配置渠道配置（电子邮件、短信、Web 推送、应用程序内、Web、直邮、内容卡片）
   * 委派子域并设置 IP 池
   * 配置禁止列表和同意策略

2. **数据工程师**：创建数据基础\
   数据工程师构建支持个性化的数据基础架构，定义客户数据如何流入系统并在系统中流转。
   * 创建用于客户识别的身份命名空间
   * 设计 XDM 架构（用户档案、体验事件、关系型数据）
   * 设置数据集并为实时客户轮廓启用它们
   * 配置数据摄取（批处理和流处理）
   * 创建用于复杂计算的计算属性
   * 为历程配置事件和数据源

3. **开发人员**：实施技术集成\
   开发人员通过集成 SDK、发送事件和构建 API 端点，将应用程序连接到 Journey Optimizer。 这些实施使得历程能够被触发并执行。
   * 将 Mobile SDK (iOS/Android) 与推送通知设置相集成
   * 实施 Web SDK 以支持 web 体验和 web 推送通知
   * 从应用程序发送事件以触发历程
   * 构建用于外部系统集成的自定义操作端点
   * 监控自定义操作的运行状况和性能
   * 使用 Adobe Experience Platform Assurance 测试实施方案

4. **营销人员**：设计和执行客户体验\
   营销人员利用所有基础工作来构建历程、创建内容并优化所有渠道的客户体验。
   * 使用分段、CSV 上传或受众构成功能构建受众群体
   * 使用 AI 助手和模板设计个性化内容
   * 使用事件和受众触发器创建多渠道历程
   * 在发布之前使用审批工作流进行测试
   * 监控性能并根据报表洞察进行优化

**注意：**&#x200B;虽然这是常规的顺序，但可以并行开展某些活动。 例如，开发人员可以在数据工程师配置架构期间处理应用程序集成。

## 按角色快速入门

在开始阶段，每个角色都要执行根据其职责重心定制的具体任务。 完成这些初始步骤可确保更顺利地加入并与整体实施流程保持一致：

### 面向营销人员 {#for-marketers}

作为营销人员或业务从业者，您负责设计客户历程，以便在所有接触点提供个性化、情境化的体验。 您将在统一的界面中工作，从头到尾完成整个用例的实施。

**您将使用的关键功能：**

* **历程编排**：打造实时、一对一的客户参与体验，让每个人按自己的节奏推进，由跨渠道的行为或事件触发。 使用统一操作活动执行所有渠道操作，使用内容决策活动将优惠集成到历程中，并使用历程代理 根据自然语言提示创建历程
* **营销活动编排**：通过可视化画布大规模设计和自动化复杂的多步骤批量营销活动。 非常适合品牌发起的营销活动，如季节性促销、产品发布和基于客户的沟通。 利用多实体细分，通过关联客户数据与相关实体（帐户、购买记录、预订信息），创建精准的受众群体 使用波次发送以受控批次发送消息
* **现代消息设计器**：通过拖放界面设计和个性化电子邮件及移动端消息。 编辑现成的模板以加速产品上市时间
* **决策管理**：在集中式库中创建和管理优惠、资格规则和其他组件，这些组件可以嵌入电子邮件和客户接触点中。 使用 Decisioning 实现推送和短信个性化
* **资产管理**：访问完全内置于 Journey Optimizer 中的 Adobe Experience Manager Assets Essentials，以实现简化的资产访问与交付
* **受众定义**：借助关系型查询即时优化，按需构建受众群体，而发送前预览功能可确保受众数量精准无误
* **AI/ML 服务**：利用发送时间优化和预测性参与度得分来定位高价值客户并将客户流失风险降至最低
* **投放控制**：使用免打扰时间（基于时间的排除）和冲突管理来尊重客户偏好并防止过度通信

**从以下开始：**&#x200B;使用案例模板和向导，轻松创建并部署新的客户历程。 使用历程代理根据自然语言提示创建历程。

[营销人员入门 →](path/marketer.md)

### 对于数据工程师 {#for-data-engineers}

作为数据架构师或工程师，您需要设置并维护客户轮廓数据及其他数据源，为 Journey Optimizer 编排的体验提供支持。

**主要职责：**

* **客户轮廓数据**：将客户轮廓数据和业务数据建模为方案，构建统一的 360 度客户视图
* **关系型数据建模**：设计关系型架构以实现多实体分段 – 将客户数据与相关实体（如帐户、购买记录、订阅信息和预订数据）关联，从而灵活创建受众群体
* **源连接器**：配置源连接器，将来自 web、CRM、离线数据及其他来源的数据摄取到 Adobe Experience Platform 中
* **身份解析**：设置身份命名空间，以持续更新用户轮廓，并实时将客户移入或移出细分群体及历程
* **数据源**：配置数据源以实时监听客户历程中的外部信号
* **轮廓管理**：启用实时客户轮廓的数据集以提升个性化体验
* **数据质量**：监控数据摄取，确保所有数据顺利流入 Journey Optimizer

**初始步骤：**&#x200B;查看[数据管理快速入门](../data/gs-data.md)概述，了解架构、数据集、身份标识和完整的数据设置清单。 然后，建立首个客户档案架构模型，并配置源连接器以开始摄取数据。

[数据工程师快速入门 →](path/data-engineer.md)

### 面向管理员 {#for-administrators}

作为管理员，您需要设置 Journey Optimizer 环境，以确保您的团队能够高效、安全地开展工作。

**主要职责：**

* **沙盒**：创建和管理沙盒，为不同的用户组（开发、测试、生产）分隔数据和历程
* **用户管理**：设置用户组和权限，以控制对不同功能的访问权限
* **渠道设置**：配置投放渠道和消息预设，确保通过 Journey Optimizer 投放的消息和资产保持品牌一致性
* **安全和管理**：应用对象级访问控制 (OLAC)，配置同意策略并实施数据治理策略
* **可投放性**：委派子域名，需要时将子域名迁移到自定义委派，创建 IP 池，管理禁止列表和允许列表
* **历程配置**：为团队设置历程元素和配置
* **渠道配置**：根据需要配置 Web 推送通知、直邮和消息导出（电子邮件/短信）

**起步事项：**&#x200B;配置沙盒和用户权限，然后设置首个渠道配置和消息预设。

[管理员入门 →](path/administrator.md)

### 面向开发人员 {#for-developers}

实施技术集成，将 Journey Optimizer 连接到您的应用程序。

**主要职责：**

* 集成 Adobe Experience Platform Mobile SDK (iOS/Android)
* 实施 Web SDK 以支持 web 体验和 web 推送通知
* 配置推送通知凭据和证书
* 从应用程序发送事件以触发历程
* 构建可供 Journey Optimizer 通过自定义操作调用的 API 端点
* 为 web、移动端及其他界面实施基于代码的体验
* 使用 Adobe Experience Platform Assurance 测试和调试实施方案
* 使用 Journey Optimizer API 实现程序化访问

**从以下开始：**&#x200B;集成 Mobile 或 Web SDK，然后实施第一个事件以触发历程。

[开发人员入门 →](path/developer.md)

## 跨角色协作

成功的 Journey Optimizer 实施需要所有角色之间的协作。 每个角色均需与其他角色协作，以提供无缝的客户体验：

>[!BEGINTABS]

>[!TAB 管理员]

**管理员**&#x200B;通过管理访问权限和配置，为所有团队提供支持。 他们与以下角色协作：

* **数据工程师**：授予数据管理权限、批准沙盒访问权限、协调治理策略
* **开发人员**：提供 API 凭据，设置测试环境，批准渠道配置
* **营销人员**：为历程/营销活动分配权限，配置渠道，支持测试环境

>[!TAB 数据工程师]

**数据工程师**&#x200B;为所有人提供数据基础。 他们与以下角色协作：

* **管理员**：请求数据管理权限、协调治理和保留策略
* **开发人员**：提供 XDM 架构和事件结构，定义事件负载格式，测试数据摄取
* **营销人员**：创建用于个性化的计算属性，构建受众群体，配置关系型架构

>[!TAB 开发人员]

**开发人员**&#x200B;实施技术集成，为历程提供动力。 他们与以下角色协作：

* **数据工程师**：获取 XDM 架构和事件结构，协调数据收集要求，测试事件传递
* **管理员**：提供 API 规范，请求权限和凭据，协调测试策略
* **营销人员**：了解事件触发器，实施跟踪，支持历程测试，排查问题

>[!TAB 营销人员]

**营销人员**&#x200B;设计客户体验并提供反馈。 他们与以下角色协作：

* **数据工程师**：请求计算属性，协调受众需求，提供数据质量反馈
* **开发人员**：协调事件触发器，测试实施，验证跟踪
* **管理员**：请求渠道配置，确认功能访问权限，协调启用事宜

>[!ENDTABS]

**最佳实践：**&#x200B;定期召开跨职能会议，以协调优先级、共享进展并解决跨团队障碍。
