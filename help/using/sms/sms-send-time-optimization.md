# 针对移动消息传送的发送时间优化 {#sms-send-time-optimization}

>[!AVAILABILITY]
>
>针对移动消息传送（短信、RCS和WhatsApp）的发送时间优化功能自2026年下半年起可用，适用于历程和促销活动。

## 概述 {#overview}

移动消息传送的发送时间优化(STO)功能通过使用人工智能驱动的洞察来确定每个用户档案的最佳交付时间，使营销人员能够超越“批量处理并爆炸”计划。 Adobe Journey Optimizer不会一次向整个受众发送消息，而是分析每个用户档案的历史参与模式并预测该个人何时最有可能打开、单击或响应短信、RCS或WhatsApp消息。

通过在预测参与度最高的时刻发送消息，STO可帮助提高打开率、点进率和总体营销活动ROI — 无需按时区或行为同类群组手动细分受众。 SMS、RCS和WhatsApp渠道均支持用于移动消息传递的STO，并且在历程和Campaign执行上下文中均可用。

## 先决条件 {#prerequisites}

在为移动消息启用发送时间优化之前，请验证以下各项：

- 您的组织已针对Adobe Journey Optimizer以及至少一个SMS、RCS或WhatsApp渠道进行了配置。
- Adobe Experience Platform (AEP)中的目标受众具有足够数量的历史移动参与数据，包括发送事件、打开、链接点击和回复。 AI模型需要先前的交互历史才能生成可靠的个人资料预测。
- 相关渠道表面（短信、RCS或WhatsApp）已配置并在您的AJO实例中处于活动状态。 有关设置说明，请参阅[配置短信渠道界面](../sms/sms-configuration.md)。
- 对于基于历程的用例，必须设计历程，使消息操作节点不受上游等待或超时与STO投放窗口冲突的事件节点的约束。

>[!NOTE]
>
>历史参与数据不足的用户档案会回退到您在配置期间定义的默认发送时间。 STO预测是在个人用户档案级别生成和评分的。

## 工作原理 {#how-it-works}

用于移动消息的STO依赖于专门构建的人工智能模型，该模型处理每个用户档案的历史参与信号以预测最佳投放窗口。 以下步骤描述了端到端流程。

### &#x200B;1. 数据引入和模型训练

AI模型会持续摄取存储在Adobe Experience Platform中的移动参与事件，包括消息打开时间戳、链接点击事件、回复时间和历史投放记录。 这些信号构成了用于学习每个用户档案的行为模式的训练数据 — 例如首选交互小时数、一周中的某天趋势以及跨时区的响应性。 该模型在滚动基础上进行重新训练，以保持对参与行为变化的敏感性。

### &#x200B;2. 每个用户档案预测评分

一旦训练，该模型就会对目标受众中的每个用户档案进行评分，并产生一个最佳发送时间窗口。 此预测将作为计算属性写入AEP中的配置文件，在执行时可供历程和营销活动使用，而无需在消息分派期间进行额外的API调用或实时查找。

### &#x200B;3. 历程运行时计划

当包含启用了STO的SMS、RCS或WhatsApp操作节点的历程上线时，历程运行时将在配置文件到达操作节点时读取每个合格配置文件的预测发送时间属性。 消息被保存在所配置的优化窗口内，并在预测的最佳时间到达时发送。 如果预测时间已经过去或超出窗口，则会应用您配置的回退行为。

### &#x200B;4. 营销活动发送分发

对于Campaigns，STO会根据每个用户档案的预测跨受众分发发送，而不是发出单个批量发送。 AJO在营销活动的配置发送窗口内交错投放，遵循每个用户档案的预测最佳时间，同时保持在您定义的窗口边界内。

>[!NOTE]
>
>如果用户档案的预测最佳时间落在配置的发送窗口之外，则消息将在最近的边界（窗口的开始或结束，以最接近预测的时间为准）发送。

## 配置发送时间优化 {#configure}

### 在营销活动中启用STO {#configure-campaign}

1. 在Journey Optimizer中，导航到&#x200B;**营销活动**&#x200B;并创建新营销活动或打开现有草稿。
2. 选择&#x200B;**SMS**、**RCS**&#x200B;或&#x200B;**WhatsApp**&#x200B;作为渠道，并完成受众和消息内容步骤。
3. 在&#x200B;**计划**&#x200B;部分中，选择&#x200B;**发送时间优化**，而不是固定发送日期和时间。
4. 使用&#x200B;**发送时间优化**&#x200B;切换启用该功能。
5. 配置&#x200B;**发送窗口**：设置允许AJO传递消息的开始和结束时间。 此窗口必须至少跨越1小时，并且最多可以扩展24小时。
6. 为参与历史记录不足以生成预测的用户档案定义&#x200B;**回退发送时间**。 您可以选择在窗口打开时立即发送，或者在窗口内的固定时间发送。
7. 完成频率封顶、审查和激活步骤，然后激活营销活动。

### 在历程中启用STO {#configure-journey}

1. 在历程画布中打开或创建历程。
2. 添加或选择&#x200B;**SMS**、**RCS**&#x200B;或&#x200B;**WhatsApp**&#x200B;操作节点。
3. 在操作节点配置面板中，展开&#x200B;**发送时间**&#x200B;设置。
4. 将&#x200B;**发送时间优化**&#x200B;切换为启用状态。
5. 设置&#x200B;**优化窗口**：运行时在等待预测的最佳投放时间时可以保留消息的最长持续时间（以小时为单位）。 默认时段为6小时；最长时段为24小时。
6. 为没有预测数据的用户档案配置&#x200B;**回退行为** — 在用户档案进入节点时立即发送或等到下一个可用预测窗口。
7. 保存节点配置并发布历程。

>[!NOTE]
>
>当STO在历程操作节点上处于活动状态时，用户档案的有效交付时间可能与用户档案进入节点的时间有所不同，最长可达配置的优化窗口的完整长度。 在设计上游等待节点并设置历程级别超时以防止过早退出旅程时，需要考虑此延迟。

## 护栏和限制 {#guardrails}

- STO仅适用于出站SMS、RCS和WhatsApp消息。 入站回复流和双向消息传递会话不受STO计划限制。
- 每个“营销活动”或“历程”操作节点支持每个启用STO的消息具有一个渠道平面。 不支持跨渠道STO协调（例如，单个节点中的SMS和WhatsApp）。
- AI模型要求每个用户档案至少30天的历史移动参与数据才能生成预测。 低于此阈值的配置文件使用配置的回退发送时间。
- STO与频率上限规则交互。 如果受限用户档案的预测投放窗口与上限限制冲突，则根据上限规则抑制消息，并且不会重新计划到以后的窗口。
- 始终强制实施同意标记、选择退出记录和全局禁止列表，而不考虑STO计划。 为优化投放而保留的邮件在发送时仍受同意检查。
- STO不适用于业务或法规要求需要立即投放的交易式消息。
