# 历程类型：选择正确的类型 {#journey-types-selection}

>[!BEGINSHADEBOX]

**在此页面上：**&#x200B;了解四种AJO旅程类型 — 单一事件、读取受众、受众资格和业务事件 — 并找出哪种类型适合您的用例。

>[!ENDSHADEBOX]

[!DNL Adobe Journey Optimizer]支持四种历程类型，每种历程类型针对不同类型的触发器和业务方案而设计。 了解这些差异有助于您从一开始就构建正确的体验。

## 四种历程类型 {#journey-types}

>[!BEGINTABS]

>[!TAB 单一事件历程]

**何时使用：**&#x200B;事件触发的实时体验

**当发生特定操作（购买、应用程序登录、表单提交）时，将单独触发**&#x200B;单一事件历程。 用户档案一次实时输入一个用户档案，因此非常适合用于即时、行为驱动的响应。

**非常适合：**&#x200B;购物车放弃恢复、新成员载入、有人订阅时的欢迎电子邮件和登录后个性化。

➡️ [了解事件](../event/about-events.md) | [发送给订阅者的消息用例](message-to-subscribers-uc.md) | [创建您的第一个历程](journey-gs.md)

>[!TAB 读取受众历程]

**何时使用：**&#x200B;预定促销活动到受众区段

**读取受众历程**&#x200B;从[!DNL Adobe Experience Platform]受众开始，并同时向所有用户档案批量发送消息。 此历程类型非常适合于定期的大规模通信。 对定期历程使用&#x200B;**增量读取**&#x200B;选项，以仅处理自上次执行以来加入受众的用户档案，而不是每次重新处理完整受众。

**非常适合：**&#x200B;每月新闻稿、针对细分市场的促销活动、产品公告、定期重新参与系列和季节性营销活动。

➡️ [了解读取受众](read-audience.md) | [受众入门](../audience/about-audiences.md) | [创建您的第一个历程](journey-gs.md)

>[!TAB 受众资格历程]

**何时使用：**&#x200B;对受众成员资格更改的实时响应

当配置文件符合（或退出）特定受众的资格时，**受众资格历程**&#x200B;会触发。 用户档案在满足标准时单独输入，这样可在客户行为发生更改时立即参与。 使用&#x200B;**流式评估**&#x200B;受众 — 这是此活动唯一支持的受众类型。

>[!CAUTION]
>
>从&#x200B;**2026年8月**&#x200B;开始，无法发布在受众资格节点中使用批次受众的历程。 [了解如何迁移您的历程](aq-batch-audiences-migration.md)

**非常适合：** VIP层级升级通知、首次购买庆祝消息、客户流失风险警报和忠诚度生命周期阶段过渡。

➡️ [了解受众资格](audience-qualification-events.md) | [创建受众](../audience/creating-a-segment-definition.md) | [创建您的第一个历程](journey-gs.md)

>[!TAB 商业活动历程]

**何时使用：**&#x200B;影响多个客户的业务情况

**业务事件历程**&#x200B;由同时影响多个配置文件的业务级事件（股票更新、价格变动）触发。 在内部，业务事件触发器之后始终有一个读取受众步骤，用于摄取相关配置文件 — 因此配置文件输入遵循读取受众吞吐量规则，而不是单一事件吞吐量。

**非常适合：**&#x200B;向感兴趣的客户发出低库存警报、快闪销售公告、降价通知和产品补货警报。

➡️ [了解业务活动](../event/about-creating-business.md) | [条目管理](entry-management.md) | [创建您的第一个历程](journey-gs.md)

>[!ENDTABS]

## 您应该使用哪种类型？ {#decision-guide}

答案通常归结为一个问题：*什么开始了历程？*

如果&#x200B;**客户执行了特定操作** — 放弃购物车、注册、购买 — 请使用&#x200B;**单一事件历程**。 它会在操作发生时立即触发，一次触发一个配置文件。

如果您希望&#x200B;**按计划**（每月新闻稿、季节性促销活动、定期重新参与系列）联系受众，请使用&#x200B;**读取受众历程**。 您可以定义受众和时间；AJO会一次处理每个人。

如果要在客户达到里程碑&#x200B;**时响应** — 加入忠诚度等级、达到流失风险阈值、完成首次购买 — 请使用&#x200B;**受众资格历程**。 当流受众成员资格发生更改时，它会立即触发，而不是按固定计划触发。

如果你的企业中有某些东西发生了变化&#x200B;**，并且这些变化同时影响了多个客户（股价下跌、价格变动、销售开始），请使用**&#x200B;业务事件历程&#x200B;**。**

>[!TIP]
>
>**不确定从何处开始？** 对于行为触发的体验，大多数团队以&#x200B;**单一事件**&#x200B;开头，对于营销活动，以&#x200B;**读取受众**&#x200B;开头。 这两者涵盖了大多数用例。

| 您的目标 | 推荐的历程类型 | 原因 |
|-----------|--------------------------|-----|
| 恢复放弃的购物车 | 单一事件 | 对个人行为的即时响应 |
| 每月向订阅者发送新闻稿 | 读取受众 | 计划的批处理通信 |
| 当客户达到VIP状态时通知客户 | 受众资格筛选 | 实时响应流受众进入 |
| 提醒客户观看项目时库存过低 | 业务事件 | 业务状况影响多个客户 |
| 欢迎新应用程序用户 | 单一事件或受众资格 | 注册事件（单一事件）或进入新用户流受众（受众资格） |
| 重新吸引不活跃客户（定期、已计划） | 读取受众 | 针对非活动受众运行重复批处理 |
| 目标区段的季节性提升 | 读取受众 | 计划的面向受众的营销活动 |
| 快闪销售公告 | 业务事件 | 业务决策影响多个客户 |
| 客户达到金会员忠诚度等级后立即做出反应 | 受众资格筛选 | 流受众，实时个人条目 |

## 功能可用性参考 {#feature-compatibility}

所有历程类型都支持完整的AJO渠道集（电子邮件、推送、短信、应用程序内、Web、内容卡）、核心编排活动（等待、条件、自定义操作）、测试模式、试运行和发送时间优化。 下表仅显示不同类型的功能。

>[!NOTE]
>
>跳转活动限制：以读取受众或受众资格活动开始的历程不能包含跳转活动，也不能作为另一历程中的跳转活动的目标。
>
>读取受众活动，因为历程条目仅在&#x200B;**读取受众**&#x200B;和&#x200B;**业务事件**&#x200B;历程中可用 — 无法将其添加到单一事件或受众资格条目历程。

| 功能 | 单一事件 | 读取受众 | 受众资格筛选 | 业务事件 |
|-----------|:-------:|:-------------:|:----------------------:|:--------------:|
| **条目** | | | | |
| 事件触发的条目 | ✅ | ❌ | ❌ | ✅ （业务事件触发历程；配置文件通过内部读取受众步骤进入） |
| 计划条目 | ❌ | ✅ | ❌ | ❌ |
| 基于受众的条目 | ❌ | ✅ （批次） | ✅ （仅流式传输） | ❌ |
| **编排** | | | | |
| 读取受众活动（历程条目） | ❌ | ✅ | ❌ | ✅ （业务事件后的自动步骤） |
| 跳转活动 | ✅ | ❌ | ❌ | ✅ |
| **配置文件管理** | | | | |
| 配置文件重新进入 | ✅可配置 | ❌默认情况下，每次执行一次（[在重复时强制重新进入](read-audience.md#schedule)可用） | ✅可配置（历程中已存在的配置文件无法重新输入相同版本） | 每个事件✅ |
| **优化** | | | | |
| 路径实验（A/B测试） | ✅ | ✅ | ✅ | ❌ |
| **高级** | | | | |
| 增量读取 | ❌ | ✅ | ❌ | ❌ |
| 最大吞吐量 | 5,000 TPS（共享组织级别的受众资格） | 每个沙盒20,000 TPS | 5,000 TPS（共享组织级别和单一事件） | 业务活动：5,000 TPS；读取受众步骤：20,000 TPS |

**图例：** ✅ =支持 | ❌ =不支持

**术语表：**

* **单一事件历程**：由特定的单个客户操作（例如，购买、登录）触发的历程，其中配置文件一次输入一个实时事件。 *（产品特定）*
* **读取受众历程**：一个以Adobe Experience Platform受众开始的历程，该历程会按计划同时将消息批量发送到所有用户档案。 *（产品特定）*
* **受众资格历程**：当配置文件符合或退出特定受众区段资格时触发的历程。 需要接受流式计算的受众才能实时进入行为。 *（产品特定）*
* **业务事件历程**：由同时影响多个用户档案的业务级别事件（例如，库存更新、价格更改）触发的历程；始终与用于用户档案摄取的内部读取受众步骤配对。 *（产品特定）*
* **增量读取**：一种读取受众功能，只处理自上次执行以来加入受众的用户档案，而不是每次都处理完整的受众。 仅适用于读取受众历程。 *（产品特定）*
* **流式受众**：Adobe Experience Platform受众是连续实时评估的，而不是按计划评估的批量受众（例如，每天）。 受众资格历程实现实时进入行为所必需的。 *（产品特定）*

**护栏：**

* 增量读取仅适用于读取受众历程，不适用于单一事件、受众资格或业务事件历程
* 业务事件历程不支持路径实验（A/B测试）
* 默认情况下，读取受众历程中的用户档案重新进入限制为每次执行一次；在计划运行时使用重复时强制重新进入，以允许用户档案在下次执行时重新进入
* 读取受众活动仅在读取受众和业务事件历程中作为历程条目提供，而不是在单一事件或受众资格条目历程中提供
* 受众资格和读取受众历程不能包含跳转活动，也不能作为另一个历程中的跳转活动的目标
* 受众资格历程需要流式评估的受众。 从2026年8月开始，无法在受众资格节点中使用批次评估的受众 — 请参阅[迁移指南](aq-batch-audiences-migration.md)
* 单一事件和受众资格历程在组织级别共享了5,000个TPS吞吐量限制；读取受众历程支持每个沙盒高达20,000个TPS
* 历程中已存在的用户档案无法重新进入该历程的相同版本，无论重新进入配置如何

**术语：**

* 规范名称：单一事件历程 — 变体：事件触发的历程，单一历程
* 规范名称：读取受众历程 — 变体：批处理历程
* 规范名称：受众资格历程 — 变体：受众资格事件历程
* 规范名称：业务事件历程 — 变体：业务事件触发的历程
* 请勿混淆：“读取受众历程”≠“受众资格历程” — 读取受众会按计划批量处理所有受众成员；受众资格会实时响应单个成员资格更改（流式受众仅供立即进入）
* 请勿混淆：“单一事件历程”≠“业务事件历程” — 单一由影响一个配置文件的客户操作触发；业务事件由业务条件触发，并通过内部读取受众步骤摄取多个配置文件

**常见问题解答：**

* **问：我应该使用哪种历程类型来制作每月的新闻稿？**  — 使用读取受众历程；该历程旨在按计划同时与受众区段中的所有用户档案进行批量通信。
* **问：我应该使用哪种历程类型来恢复放弃的购物车？**  — 使用单一事件历程；当发生放弃事件时立即触发，并实时响应个人的行为。
* **问：我能否在业务事件历程中运行A/B路径实验？**  — 不支持；业务事件历程不支持路径实验。
* **问：单一事件历程与受众资格历程之间有何区别？**  — 单一事件历程由特定客户操作（例如，购买）触发；当用户档案根据流标准评估进入或退出受众区段时，将触发受众资格历程。
* **问：哪些历程类型支持增量读取？**  — 只有读取受众历程支持增量读取，而其他三种历程类型不支持。
* **问：能否将读取受众活动添加到单一事件历程？**  — 否；读取受众活动仅在读取受众和业务事件历程中作为历程条目提供。
* **问：能否在读取受众历程中使用跳转活动？**  — 否；以读取受众或受众资格活动开始的历程不能包含跳转活动，也不能作为来自另一历程的跳转的目标。
* **问：我能否欢迎新应用程序用户参加受众资格鉴定历程？**  — 可以，如果条目由流受众驱动（例如，当用户档案加入新用户区段时）；注册单一事件历程也是常见模式。
* **问：我的受众资格历程未实时触发。 为什么？**  — 受众资格历程需要流式评估的受众。 弃用使用批量计算的受众，并将从2026年8月起阻止使用该受众。 [请参阅迁移指南](aq-batch-audiences-migration.md)
* **问：单一事件与读取受众历程之间的吞吐量差异是多少？**  — 单一事件历程与组织级别的受众资格历程具有5,000个TPS限制。 读取每个沙盒支持高达20,000个TPS的受众历程，使其更适合大规模批量活动。
