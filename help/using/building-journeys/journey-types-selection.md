# 历程类型和选择指南 {#journey-types-selection}

>[!BEGINSHADEBOX]

**在此页面上：**&#x200B;了解如何比较四种历程类型（单一事件、读取受众、受众资格和业务事件），并使用决策指南和功能兼容性矩阵为您的用例选择合适的历程类型。

>[!ENDSHADEBOX]

[!DNL Adobe Journey Optimizer]支持四种历程类型，每种历程类型针对不同的进入机制和业务方案而设计。 本指南可帮助您了解这些差异，为您的用例选择正确的类型。

>[!NOTE]
>
>不确定选择哪种类型？ 对于基于事件的体验，从&#x200B;**单一事件历程**&#x200B;开始，对于计划的营销活动，从&#x200B;**读取受众历程**&#x200B;开始 — 这些涵盖最常见的用例。

## 历程类型概述 {#journey-types}

>[!BEGINTABS]

>[!TAB 单一事件历程]

**何时使用：**&#x200B;事件触发的实时体验

**当发生特定操作（购买、应用程序登录、表单提交）时，将单独触发**&#x200B;单一事件历程。 用户档案一次实时输入一个用户档案，因此非常适合用于即时、行为驱动的响应。

**非常适合：**&#x200B;购买后的订单确认、有人订阅时的欢迎电子邮件、密码重置通知以及登录后个性化。

➡️ [了解事件](../event/about-events.md) | [发送给订阅者的消息用例](message-to-subscribers-uc.md)

>[!TAB 读取受众历程]

**何时使用：**&#x200B;预定促销活动到受众区段

**读取受众历程**&#x200B;从[!DNL Adobe Experience Platform]受众开始，并同时向所有用户档案批量发送消息。 此历程类型非常适合于定期的大规模通信。 对定期历程使用&#x200B;**增量读取**&#x200B;选项，以仅处理自上次执行以来加入受众的用户档案，而不是每次重新处理完整受众。

**非常适合：**&#x200B;每月新闻稿、针对细分市场的促销活动、产品公告、定期重新参与系列和季节性营销活动。

➡️ [了解读取受众](read-audience.md) | [受众入门](../audience/about-audiences.md)

>[!TAB 受众资格历程]

**何时使用：**&#x200B;对受众成员资格更改的实时响应

当配置文件符合（或退出）特定受众的资格时，**受众资格历程**&#x200B;会触发。 用户档案在满足标准时单独输入，这样可在客户行为发生更改时立即参与。 对于实时条目行为，受众必须为&#x200B;**流式评估**；批量评估的受众仅在下一个评估窗口（最多24小时）触发条目。

**非常适合：** VIP层级升级通知、首次购买庆祝消息、客户流失风险警报和忠诚度生命周期阶段过渡。

➡️ [了解受众资格](audience-qualification-events.md) | [创建受众](../audience/creating-a-segment-definition.md)

>[!TAB 商业活动历程]

**何时使用：**&#x200B;影响多个客户的业务情况

**业务事件历程**&#x200B;由同时影响多个配置文件的业务级事件（股票更新、价格变动）触发。 在内部，业务事件触发器之后始终有一个读取受众步骤，用于摄取相关配置文件 — 因此配置文件输入遵循读取受众吞吐量规则，而不是单一事件吞吐量。

**非常适合：**&#x200B;向感兴趣的客户发出低库存警报、快闪销售公告、降价通知和产品补货警报。

➡️ [了解业务活动](../event/about-creating-business.md) | [条目管理](entry-management.md)

>[!ENDTABS]

## 决策指南：选择您的历程类型 {#decision-guide}

使用下表将您的目标与正确的历程类型相匹配。 对于大多数新用户，**单一事件**&#x200B;或&#x200B;**读取受众**&#x200B;历程涵盖了大多数用例。

| 您的目标 | 推荐的历程类型 | 原因 |
|-----------|--------------------------|-----|
| 购买后发送订单确认 | 单一事件 | 对单个操作的即时响应 |
| 每月向订阅者发送新闻稿 | 读取受众 | 计划的批处理通信 |
| 当客户达到VIP状态时通知客户 | 受众资格筛选 | 实时响应流受众进入 |
| 提醒客户观看项目时库存过低 | 业务事件 | 业务状况影响多个客户 |
| 欢迎新应用程序用户 | 单一事件 | 由注册事件触发 |
| 重新吸引不活跃客户（定期、已计划） | 读取受众 | 针对非活动受众运行重复批处理 |
| 目标区段的季节性提升 | 读取受众 | 计划的面向受众的营销活动 |
| 快闪销售公告 | 业务事件 | 业务决策影响多个客户 |
| 客户达到金会员忠诚度等级后立即做出反应 | 受众资格筛选 | 流受众，实时个人条目 |

## 历程类型详细比较 {#journey-types-comparison}

| 长宽比 | 单一事件历程 | 读取受众历程 | 受众资格历程 | 业务事件历程 |
|--------|------------------------|------------------------|--------------------------------|------------------------|
| **进入机制** | 单个事件触发器 | 计划批次 | 实时流受众会员资格更改 | 业务级事件+读取受众步骤 |
| **进入计时** | 实时发生 | 已计划（一次性或周期性） | 在进行资格鉴定时实时（流受众）；延迟接受批量评估的受众 | 实时触发器；配置文件摄取遵循读取受众吞吐量 |
| **配置文件条目** | 一次一个 | 一次全部（批次） | 一次一个 | 通过内部读取受众步骤访问多个配置文件 |
| **触发器源** | 客户操作（购买、点击、登录） | 基于时间的计划 | 进入或退出受众成员资格 | 经营条件（库存、价格） |
| **最适合** | 事务型消息、行为响应 | 营销活动、新闻稿、定期项目 | 忠诚度计划，生命周期阶段过渡 | 库存警报、促销活动、业务状况 |
| **使用时间** | 对所需的单个操作立即做出响应 | 按时访问大量受众区段 | 实时响应客户状态更改 | 业务事件同时影响多个客户 |
| **示例** | 订单确认、密码重置 | 每月新闻稿，季节性营销活动 | VIP升级，流失风险警报 | 低库存警报，闪售，价格下跌 |
| **重新进入** | 可配置 | 每次执行一次 | 可根据资格事件进行配置；历程中已有的用户档案无法重新输入相同版本 | 同一事件可能会影响多个配置文件 |
| **最大吞吐量** | 5,000 TPS（共享组织级别的受众资格） | 每个沙盒20,000 TPS | 5,000 TPS（共享组织级别和单一事件） | 业务活动：5,000 TPS；读取受众步骤：20,000 TPS |
| **数据要求** | 包含触发器数据的事件架构 | [!DNL Adobe Experience Platform]受众 | 流式受众（实时录入所需）；支持批量受众，但会延迟录入 | 业务事件架构 |

**术语表：**

* **单一事件历程**：由特定的单个客户操作（例如，购买、登录）触发的历程，其中配置文件一次输入一个实时事件。 *（产品特定）*
* **读取受众历程**：一个以Adobe Experience Platform受众开始的历程，该历程会按计划同时将消息批量发送到所有用户档案。 *（产品特定）*
* **受众资格历程**：当配置文件符合或退出特定受众区段资格时触发的历程。 需要接受流式计算的受众才能实时进入行为。 *（产品特定）*
* **业务事件历程**：由同时影响多个用户档案的业务级别事件（例如，库存更新、价格更改）触发的历程；始终与用于用户档案摄取的内部读取受众步骤配对。 *（产品特定）*
* **增量读取**：一种读取受众功能，只处理自上次执行以来加入受众的用户档案，而不是每次都处理完整的受众。 仅适用于读取受众历程。 *（产品特定）*
* **流式受众**：Adobe Experience Platform受众是连续实时评估的，而不是按计划评估的批量受众（例如，每天）。 受众资格历程实现实时进入行为所必需的。 *（产品特定）*

**护栏：**

* 增量读取仅适用于读取受众历程，不适用于单一事件、受众资格或业务事件历程
* 业务事件历程不支持路径实验（A/B测试）
* 读取受众历程中的用户档案重新进入限制为每个执行一次
* 受众资格和读取受众历程不能包含跳转活动，也不能作为另一个历程中的跳转活动的目标
* 受众资格历程需要实时输入的流评估受众；批量评估受众会导致最多24小时的输入延迟
* 单一事件和受众资格历程在组织级别共享了5,000个TPS吞吐量限制；读取受众历程支持每个沙盒高达20,000个TPS
* 历程中已存在的用户档案无法重新进入该历程的相同版本，无论重新进入配置如何

**术语：**

* 规范名称：单一事件历程 — 变体：事件触发的历程，单一历程
* 规范名称：读取受众历程 — 变体：批处理历程、区段触发历程、读取区段历程
* 规范名称：受众资格历程 — 变体：受众资格事件历程
* 规范名称：业务事件历程 — 变体：业务事件触发的历程
* 请勿混淆：“读取受众历程”≠“受众资格历程” — 读取受众会按计划批量处理所有受众成员；受众资格会实时响应单个成员资格更改（流式受众仅供立即进入）
* 请勿混淆：“单一事件历程”≠“业务事件历程” — 单一由影响一个配置文件的客户操作触发；业务事件由业务条件触发，并通过内部读取受众步骤摄取多个配置文件

**常见问题解答：**

* **问：我应该使用哪种历程类型来制作每月的新闻稿？**  — 使用读取受众历程；该历程旨在按计划同时与受众区段中的所有用户档案进行批量通信。
* **问：购买后哪种历程类型处理订单确认？**  — 使用单一事件历程；它提供对单个客户操作的即时实时响应。
* **问：我能否在业务事件历程中运行A/B路径实验？**  — 不支持；业务事件历程不支持路径实验。
* **问：单一事件历程与受众资格历程之间有何区别？**  — 单一事件历程由特定客户操作（例如，购买）触发；当用户档案根据流标准评估进入或退出受众区段时，将触发受众资格历程。
* **问：哪些历程类型支持增量读取？**  — 只有读取受众历程支持增量读取，而其他三种历程类型不支持。
* **问：能否在读取受众历程中使用跳转活动？**  — 否；以读取受众或受众资格活动开始的历程不能包含跳转活动，也不能作为来自另一历程的跳转的目标。
* **问：我的受众资格历程未实时触发。 为什么？**  — 受众资格历程需要流式评估的受众。 如果对受众进行批量评估（例如，每日快照），则条目将延迟到下一次评估时段，这可能长达24小时。
* **问：单一事件与读取受众历程之间的吞吐量差异是多少？**  — 单一事件历程与组织级别的受众资格历程具有5,000个TPS限制。 读取每个沙盒支持高达20,000个TPS的受众历程，使其更适合大规模批量活动。
