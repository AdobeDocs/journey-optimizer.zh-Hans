# 历程类型和选择指南 {#journey-types-selection}

[!DNL Adobe Journey Optimizer]支持四种历程类型，每种历程类型针对不同的进入机制和业务方案而设计。 本指南可帮助您了解这些差异，为您的用例选择正确的类型。

## 历程类型概述 {#journey-types}

>[!BEGINTABS]

>[!TAB 单一历程]

**何时使用：**&#x200B;事件触发的实时体验

当发生特定操作（购买、应用程序登录、表单提交）时，**单独触发单一历程**。 用户档案一次实时输入一个用户档案，因此非常适合用于即时、行为驱动的响应。

**非常适合：**&#x200B;购买后的订单确认、有人订阅时的欢迎电子邮件、浏览触发的购物车放弃和密码重置通知。

➡️ [了解事件](../event/about-events.md) | [发送给订阅者的消息使用案例](message-to-subscribers-uc.md)

>[!TAB 读取受众历程]

**何时使用：**&#x200B;预定促销活动到受众区段

**读取受众历程**&#x200B;从[!DNL Adobe Experience Platform]受众开始，并同时向所有用户档案批量发送消息。 此历程类型非常适合于定期的大规模通信。

**非常适合：**&#x200B;每月新闻稿、针对细分市场的促销活动、产品公告和季节性营销活动。

➡️ [了解读取受众](read-audience.md) | [开始使用受众](../audience/about-audiences.md)

>[!TAB 受众资格历程]

**何时使用：**&#x200B;对受众成员资格更改的实时响应

当配置文件符合（或退出）特定受众的资格时，**受众资格历程**&#x200B;会触发。 用户档案在满足标准时实时单独输入，从而在客户行为发生更改时实现即时参与。

**非常适合：** VIP层升级通知、客户不活跃时重新参与、首次购买庆祝消息以及客户移动时的地理定位。

➡️ [了解受众资格](audience-qualification-events.md) | [创建受众](../audience/creating-a-segment-definition.md)

>[!TAB 商业活动历程]

**何时使用：**&#x200B;影响多个客户的业务情况

**业务事件历程**&#x200B;由同时影响多个配置文件的业务级事件（库存更新、天气警报、价格变化）触发。 这些策略应对更广泛的业务环境，而不是单个行动。

**非常适合：**&#x200B;面向感兴趣客户的低库存警报、闪电销售公告、基于天气的促销活动、降价通知和产品补货警报。

➡️ [了解业务活动](../event/about-creating-business.md) | [条目管理](entry-management.md)

>[!ENDTABS]

## 决策指南：选择历程类型 {#decision-guide}

按照此决策树为您的用例选择正确的历程类型：

### 步骤1：触发历程的是什么？

* **客户执行特定操作**（购买、点击、登录）→转到步骤2
* **时间/计划** （在特定时间或定期发送）→**使用读取受众历程**
* **客户状态更改** （加入/离开区段）→转到步骤3
* **业务状况**（库存水平、价格变化、天气）→**使用业务事件历程**

### 步骤2：单个客户操作触发器

* **是否需要即时响应？**
   * 是→**使用单一历程**
   * 无→考虑具有计划执行的读取受众历程

### 步骤3：客户状态更改

* **客户进入或退出区段时需要响应？**
   * 是→**使用受众资格历程**
   * 否，仅当进入带事件→考虑单一历程或带受众过滤器的读取受众时

### 按用例快速选择

| 您的目标 | 推荐的历程类型 | 原因 |
|-----------|------------------------|-----|
| 购买后发送订单确认 | 单一 | 对单个操作的即时响应 |
| 每月向订阅者发送新闻稿 | 读取受众 | 计划的批处理通信 |
| 当客户达到VIP状态时通知客户 | 受众资格筛选 | 对状态变化的实时响应 |
| 提醒客户观看项目时库存过低 | 业务事件 | 业务状况影响多个客户 |
| 欢迎新应用程序用户 | 单一 | 由注册事件触发 |
| 重新吸引非活跃客户 | 受众资格筛选 | 对不活动区段条目做出响应 |
| 目标区段的季节性提升 | 读取受众 | 计划的面向受众的营销活动 |
| 快闪销售公告 | 业务事件 | 业务决策影响多个客户 |

>[!NOTE]
>
>不确定选择哪种类型？ 对于基于事件的体验，从&#x200B;**单一历程**&#x200B;开始，对于计划的营销活动，从&#x200B;**读取受众历程**&#x200B;开始 — 这些涵盖最常见的使用案例。

## 历程类型详细比较 {#journey-types-comparison}

使用此表可快速比较历程类型并根据用例选择正确的类型：

| 长宽比 | 单一历程 | 读取受众历程 | 受众资格历程 | 业务事件历程 |
|--------|------------------|------------------------|--------------------------------|------------------------|
| **进入机制** | 单个事件触发器 | 计划批次 | 实时受众会员资格更改 | 业务级事件 |
| **进入计时** | 实时发生 | 已计划（一次性或周期性） | 在进行资格鉴定时实时 | 业务事件触发时的实时 |
| **配置文件条目** | 一次一个 | 一次全部（批次） | 一次一个 | 同时使用多个配置文件 |
| **触发器源** | 客户操作（购买、点击、登录） | 基于时间的计划 | 受众成员资格（登入/退出） | 业务状况（库存、天气、价格） |
| **最适合** | 事务型消息、行为响应 | 营销活动、新闻稿 | 忠诚度计划、生命周期阶段 | 库存警报、促销活动、业务状况 |
| **使用时间** | 对所需的单个操作立即做出响应 | 按时访问大量受众区段 | 响应客户状态更改 | 业务事件影响多个客户 |
| **示例** | 订单确认、密码重置 | 每月新闻稿，季节性营销活动 | VIP升级，不活动警报 | 低库存警报，闪售，价格下跌 |
| **重新进入** | 可配置（每个配置文件允许多个条目） | 每次执行每个用户档案都输入一次 | 可按资格事件配置 | 同一事件可能会影响多个配置文件 |
| **数据要求** | 包含触发器数据的事件架构 | [!DNL Adobe Experience Platform]受众 | 流式或批量受众 | 业务事件架构 |
