# 动态内容快速入门 {#start-dynamic-content}

>[!BEGINSHADEBOX]

**在此页面上：**&#x200B;了解动态内容如何使用根据用户档案属性、上下文事件和受众生成的条件规则来调整消息，使其适应目标用户档案。

>[!ENDSHADEBOX]

>[!CONTEXTUALHELP]
>id="ajo_conditions_list"
>title="条件"
>abstract="条件规则让您可以根据轮廓属性、上下文事件或受众，在消息中显示多个内容变体。"

动态内容允许您根据&#x200B;**条件规则**&#x200B;调整消息的内容，条件规则可由轮廓属性、上下文事件或受众组成。 条件规则是使用个性化编辑器中的可视化规则生成器创建的，您可以在其中存储这些规则以供在历程和营销活动中进一步重复使用。

可以在电子邮件Designer和个性化编辑器中利用条件规则，以&#x200B;**创建动态内容**，该内容将适应消息中定向的用户档案。

>[!TAB 概述]

**TL；DR**

本页介绍Journey Optimizer中的动态内容 — 基于用户档案属性、上下文事件和受众构建的条件规则如何允许消息适应它们定位的用户档案。

**意图**

* 了解什么是动态内容以及它与静态消息内容有何不同
* 确定动态内容的两个入口点：电子邮件Designer和个性化编辑器
* 导航到用于构建条件规则或创建动态内容的相应指南

>[!TAB 术语表]

* **动态内容**：根据条件规则调整的消息内容；根据是否满足定义的条件，向不同的用户档案显示不同的内容。 *（产品特定）*
* **条件规则**：一组用于确定用户档案接收的内容变体的标准，这些标准基于用户档案属性、上下文事件或受众构建。 *（产品特定）*
* **可视化规则生成器**：个性化编辑器中的交互式画布，用于创建和存储条件规则。 *（产品特定）*

>[!TAB 术语]

* **规范名称：**&#x200B;动态内容 — 变体：条件内容，个性化内容
* **同义词：** &quot;conditional rules&quot; = &quot;conditions&quot; （在UI中标记）
* **请勿混淆：**&#x200B;动态内容（根据条件规则调整每个配置文件的内容）≠静态内容（向所有收件人显示的相同内容，不考虑条件）

>[!TAB 常见问题解答]

**问：条件规则可以使用哪些类型的条件？**

条件规则可以从配置文件属性、上下文事件和受众成员资格中构建。

**问：条件规则可在何处用于创建动态内容？**

在电子邮件Designer（以创建组件变体）和个性化编辑器（以使用条件括起表达式块）中。 两个入口点都包含在链接的页面中。

**问：条件规则能否在历程和营销活动中重用？**

是的。 可以存储条件规则，以便在历程和营销活动中进一步重复使用。
