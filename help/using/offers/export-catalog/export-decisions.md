# 决策数据集{#decisions-dataset}

每次修改优惠时，都会更新自动生成的决策数据集。

![](../assets/dataset-activities.png)

数据集中最近成功的批处理显示在右侧。 数据集模式的层次视图显示在左窗格中。

>[!NOTE]
>
>了解如何访问[本节](../export-catalog/access-dataset.md)中优惠库每个对象的导出数据集。

决策(以前称为优惠决策)用于控制决策过程。 它指定应用于总库存的筛选器以按主题/类别缩小优惠，放置用于将库存缩小到技术上适合优惠的保留空间的优惠，并指定在合并约束取消所有可用个性化优惠的资格时的回退选项。

以下是可用于&#x200B;**[!UICONTROL Decision Object Repository - Decisions]**&#x200B;数据集(以前称为决策对象存储库 — 活动)的所有字段的列表。

## 标识符

记录的唯一标识符。

类型：字符串

## _experience（体验）

### 决策

#### 标准

定义一组决策标准，其中每个标准都包含一组约束。

类型：阵列

* **描述**

   标准描述。 它用于传达人类可读的意图，说明该标准是如何建立的、为何建立的以及它如何影响决定。

   类型：字符串

* **选项选择**

   选项选择定义选项在此上下文中的有效性/适用性。

   类型：对象

   * **描述**

      选项选择说明。 它用于传达人类可读意图，说明如何或为何构建此选项选择和/或将匹配哪些选项。

      类型：字符串

   * **选项筛选器**

      对基于标记的过滤器的引用，该过滤器使用其附加的标记匹配来自清单的选项。 该值是引用的决策规则的URI(@id)。 请参阅模式 https://ns.adobe.com/experience/decisioning/filter。

      类型：字符串

   * **用户档案约束类型**

      确定当前是否设置了任何约束以及表示约束的方式。 它可能是通过过滤器查询或通过一个或多个区段成员关系。

      类型：字符串

   * **选项列表**

      一个列表，它直接指定选项，而不评估过滤器查询。 可以指定选项列表或选项筛选规则。

      类型：阵列

      <!--Missing title under Option List? Desc = An identifier of an decision option entity. The value value refers to an `@id` property of a decision option.-->

* **版面**

   放置约束表示此标准仅适用于列出的放置。 仅当目标位置处于`xdm:placements`列表时，才会考虑选项选择。 否则，将跳过整个决策标准。 当忽略或空的“xdm:applients”列表时，将考虑任何目标放置的标准。 此处列出的位置会为选项选择强加隐式条件。 要考虑的选项必须具有目标放置的表示形式。

   类型：阵列

   * **位置标识符**

   对放置图元的参照。 该值是所引用的放置的URI(@id)。 请参阅模式 https://ns.adobe.com/experience/decisioning/placement。

   类型：字符串

* **用户档案约束**

   用户档案约束决定在此时，在此上下文中某个选项选择是否符合此用户档案标识的条件。 如果用户档案约束不需要考虑每个选项的值，即它是选项选择中的选项的不变量，则计算为“false”的用户档案约束将取消整个选项选择。 另一方面，将为选项选择的每个限定选项评估以选项为参数的用户档案约束规则。

   类型：对象

   * **描述**

      用户档案约束描述。 它用于传达人类可读的意图，说明该用户档案约束是如何构建的、为什么构建的和/或该约束将包括或排除哪个选项。

      类型：字符串

   * **合格规则**

      对决策规则的引用，该规则对给定用户档案和/或其他给定上下文XDM对象的计算结果为true或false。 该规则用于确定该选项是否符合给定用户档案。 该值是引用的决策规则的URI(@id)。 请参阅模式 https://ns.adobe.com/experience/decisioning/rule。

      类型：字符串

   * **用户档案约束类型**

      确定当前是否设置了任何约束以及表示约束的方式。 可以是规则，也可以是通过一个或多个区段成员关系。

      类型：字符串

      可能的值：“none”、“vicilityRule”、“anySegments”、“allSegments”、“rules”

      默认值：&quot;none&quot;

   * **区段标识符**

      区段的标识符

      字符串：阵列

      * **标识符**

         相关命名空间中区段的标识。

         类型：字符串

      * **命名空间**

         与`xid`属性关联的命名空间。

         类型：对象

         * **代码**

            该代码是命名空间的可读标识符，可用于请求用于标识图处理的技术命名空间标识。

            类型：字符串
      * **体验标识符**

         当存在时，此值表示跨命名空间标识符，该标识符在所有命名空间中所有命名空间范围标识符之间是唯一的。

         类型：字符串


* **排名详细信息**

   排名（优先级）。 定义在决策标准的上下文中如何确定\&quot;最佳选项\&quot;。 在满足用户档案约束的所有选定选项中，排名将决定要建议的前（或前N）个选项。

   类型：对象

   * **订单评估**

      对一个或多个决策选项的相对顺序的评估。 在具有较低序数值的任何选项上，都会选择具有较高序数值的选项。 该方法确定的值可以排序，但它们之间的距离不能测量，且不能求和或求积。 中值和模式是衡量序数数据中心趋势的唯一指标。

      类型：对象

      * **评分功能**

         对计算此决策选项的数值得分的函数的引用。 然后，将按该分数对决策选项进行排序（排名）。 此属性的值是每次使用on选项调用的函数的URI(@id)。 请参阅模式 https://ns.adobe.com/experience/decisioning/function。

         类型：字符串

      * **订单评估类型**

         指定使用哪个订单评估机制、决策选项的静态优先级、计算每个选项的数值的评分函数或接收排序列表的排名策略。

         类型：字符串

      * **排名策略**

         指对决策选项的列表进行排名的战略。 决策选项将以有序列表返回。 此属性的值是每次使用on选项调用的函数的URI(@id)。 请参阅模式 https://ns.adobe.com/experience/decisioning/rankingStrategy。

         类型：字符串
   * **优先级**

      单个决策选项相对于所有其他选项的优先级。 使用此属性，将优先选择不给定顺序函数的选项。 优先级值较高的选项在优先级较低的选项之前被选择。 如果两个或更多符合条件的期权具有最高优先级值，则以统一随机方式选择一个期权并用于决策建议。

      类型：整数

      最小值：0

      默认值：0


#### 活动结束日期和时间

决定结束日期和结束时间。 属性在http://schema.org/Action上定义了模式.org的“endTime”属性的语义。

类型：字符串

#### 回退选项

在此决策上下文中进行决策时使用的回退选项引用不限定任何常规选项（通常在应用硬约束时发生这种情况）。 该值是引用的回退选项的URI(@id)。

类型：字符串

#### 活动名称

在各种用户界面中显示的决策名称。

类型：字符串

#### 活动开始日期和时间

决策开始日期和结束时间。 属性在http://schema.org/Action上定义了模式.org的“startTime”属性的语义。

类型：字符串

## _repo

### 活动 ETag

拍摄快照时决策对象所在的修订版本。

类型：字符串
