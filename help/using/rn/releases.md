# Journey Optimizer 发布周期 {#releases}

[!DNL Adobe Journey Optimizer] 会定期更新，提供新增功能、改进和修复。本页介绍发布周期的原理以及每个发布阶段的含义，以便您轻松了解贵组织何时可以使用这些功能。

## 连续投放模型 {#continuous-delivery-model}

[!DNL Adobe Journey Optimizer] 在连续投放模型的基础上运行，支持采用可扩展的分阶段方法进行功能部署。此模型允许 Adobe 更快地提供创新，并确保在推出期间保持稳定性和性能。

>[!NOTE]
>
> 由于 [!DNL Adobe Journey Optimizer] 使用连续投放，因此新功能可能会跨组织或区域逐步显示。

作为此模型的一部分：

* 新功能和改进可在准备就绪后立即部署到生产环境。

* 在每月发布版本之间，会更新包含&#x200B;_最新更新_&#x200B;部分的&#x200B;[**发行说明**](release-notes.md)，介绍推出的新增功能和改进，以便您及时了解最新信息。

## 发布时间和频率 {#release-timing}

[!DNL Adobe Journey Optimizer] 通常保持每月一次的发布频率，通常在每月的最后一周进行部署。每月发行说明和相关文档在发布周的星期二发布。预发行说明在发布周前的星期五发布。

>[!TIP]
>
> 在每个季度末，可在月底前最多两周期待并转用版本，以便与季度时间表或相关的产品发布保持一致。

每月发布版本引入了一系列主要的新功能与修复，而持续交付方法允许在周期之间随时部署已准备好的额外更新。然后，在&#x200B;_最新更新_&#x200B;部分中相应地更新发行说明并注明发布日期。在发布之日的每月发行说明中将整合当月发布的所有更改。


## 发布路径 {#release-paths}

Journey Optimizer 中的功能根据其复杂性、相关性和范围而遵循不同的发布路径。该平台使用多个可用性标签（Beta、有限发布、正式发布），但并非每个功能都需要经过这些阶段。

常见的发布路径包括：

* **直接 GA** — 某些新增功能和改进将直接正式发布 (GA)。
* **LA → GA** – 某些功能先供部分受众使用（有限发布），然后再正式推出。
* **Beta → LA → GA** — 更多的或试验性功能将历经所有阶段进行测试和验证。
* **Beta → GA** — 某些稳定的 Beta 功能可以直接 GA，而无需中间的 LA 阶段。

>[!TIP]
>
> 如果您想提前体验 Beta 功能或有限发布功能，请联系 Adobe 代表讨论参加选项。


## 可用性标签 {#availability-labels}

| **标签** | **用途** | **可用性** | **关键说明** |
|------------|-------------|------------------|----------------|
| **Beta 版** | 早期测试和反馈收集。 | 仅限于参加 Adobe Beta 计划的部分客户或组织。 | - 不用于生产用途。<br>- 可能会在 GA 之前更改功能或设计。<br>- 反馈有助于优化最终实施。 |

| **正式发布 (GA)** | 广泛发布完全支持的功能。 | 默认为所有符合条件的组织启用。 | - 可用于生产且提供完全支持。<br> - 可能适用许可或授权。<br> - 可能逐步跨区域推出。 |


## 推出和可用性 {#rollout}

即使在 GA 发布后，也可能跨组织或区域逐步推出。如果新增功能未立即在您的环境中显示，则通常会在发布后的几天内可用。

此逐步推出可帮助 Adobe 在完成部署之前监控稳定性、性能和用户体验。


## 随时了解情况 {#staying-informed}

要了解最新信息：

* 查看&#x200B;[**最新发行说明**](release-notes.md)，了解新增功能和更新功能。
* 查看每月发布之间的&#x200B;**_最新更新_**&#x200B;部分，以便进行实时部署。
* 持续关注&#x200B;**预发行说明**（在可用时），提前了解即将推出的功能。
* 有关 Beta 或有限发布访问权限或授权的信息，请联系 Adobe 代表。

您可以订阅有关 Journey Optimizer 产品发布版本的&#x200B;**电子邮件和产品内提醒**。要进行订阅：

1. 请导航到 **Adobe Experience Cloud 首选项**。
1. 在&#x200B;**通知**&#x200B;下，找到 **Journey Optimizer**
1. 启用&#x200B;**新版本**&#x200B;应用程序内通知和电子邮件通知

## 常见问题 {#faq}

您可在下方查看 Adobe Journey Optimizer 发布周期的常见问题。

需要更多信息？使用本页底部的反馈选项提出问题，或通过 [Adobe Journey Optimizer 社区](https://experienceleaguecommunities.adobe.com/t5/adobe-journey-optimizer/ct-p/journey-optimizer?profile.language=zh-hans){target="_blank"}进行联系。

+++ Adobe Journey Optimizer 的发行计划是什么时候？

[!DNL Adobe Journey Optimizer] 通常在每月最后一周发布更新。每季度末，为了与跨解决方案或平台范围的更新保持一致，发布可能会最多提前两周。

+++

+++ 为什么在新增功能发布后没有立即看到该功能？

部分 GA 功能将逐步推出，以确保平台稳定性和性能。如果您没有立即看到某个功能，则会在为您的区域或组织完成部署后显示该功能。

+++

+++ Beta、有限发布和 GA 之间有何区别？

* **Beta**：早期测试阶段，有限的访问权限，基于反馈。
* **有限发布 (LA)**：控制推出，进行最终验证。
* **正式发布 (GA)**：完整版本，面向所有符合条件的客户。

+++

+++ 所有功能是否都会经过 Beta 和有限发布阶段？

不是。某些功能会直接作为 GA 版发布或只作为 LA 版发布，具体取决于其性质和准备情况。发布路径针对每项功能而定制，以实现灵活性、质量和稳定性之间的平衡。

+++

+++ 如何参与 Beta 或有限发布计划？

可通过邀请或联系 Adobe 代表提出访问申请。参与有助于塑造功能设计，并确保在最终版本中考虑您的用例。

+++

+++ 多久更新一次发行说明？

发行说明将持续更新。除每月发行外，每当将新增功能或改进推送到生产环境时都会更新&#x200B;_最新更新_&#x200B;部分。
